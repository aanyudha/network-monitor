# Security Policy

## Supported versions

The latest minor release on the default branch receives security fixes.

## Reporting a vulnerability

Please do not open public issues for suspected security vulnerabilities.

Instead:

1. Share a private report with maintainers.
2. Include affected version, impact, reproduction steps, and suggested mitigations if known.
3. Allow reasonable time for triage and coordinated disclosure.

## Scope notes

- Do not include real customer topology, credentials, or private IP inventory in reports.
- Proof-of-concept data should use sample addresses and sanitized logs.
